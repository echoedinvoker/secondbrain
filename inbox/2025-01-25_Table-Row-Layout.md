Let's check the documentation about the Table Row Layout:
- There is a fixed-size header (occupying 23 bytes on most machines)
- followed by an optional null bitmap, an optional object ID field, and the user data.

Let's query in the database to see if our guess is correct:

```sql
SELECT id
FROM users
WHERE username = 'Gene76';


--- OUTPUT ---
 id  
-----
 203  -- it's correct
(1 row)
```

We can keep doing the same thing from the other ItemId to get the other user data. That's how the PostgreSQL works.
