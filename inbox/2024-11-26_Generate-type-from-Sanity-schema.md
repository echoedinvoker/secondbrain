Then, we can use these types anywhere in our main project instead of building them manually.

```ts
// src/lib/types/sanity.d.ts

type DevExperience = {  // remember remove `export` keyword because this file is globally available
  _id: string
  _type: 'devExperience'
  _createdAt: string
  _updatedAt: string
  _rev: string
  jobTitle: string  // remove ? because it's required
  company: string  // remove ? because it's required
  startDate: string  // remove ? because it's required
  endDate?: string
}

```

Above, all public properties are set to have a question mark `?` by default, regardless of whether they are set as required in the schema, so we need to manually remove it from the required public properties.
